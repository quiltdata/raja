Overview

The system compiles authorization once in RAJA and enforces by subset checks in RAJEE.

 ┌──────────┐   Cedar policies    ┌─────────┐
 │   RAJA   │ ──────────────────► │   AVP   │
 │ Authority│ ◄── ALLOW/DENY ──── │ (Cedar) │
 └────┬─────┘                     └─────────┘
      │ issues RAJ / TAJ
      ▼
 ┌──────────┐   validates + routes  ┌──────────┐
 │  RAJEE   │ ─────────────────────►│    S3    │
 │ (Envoy)  │                       │ (actual) │
 └──────────┘
      ▲
      │ attaches token, routes request
 ┌──────────┐   logical path (USL)
 │  Diwan   │ ◄── developer uses boto3 normally
 └──────────┘
